# Проект по генерации и валидации данных клиентов

Этот проект содержит инструментарий для создания синтетических данных о клиентах и проведения проверок качества данных (Data Quality). Код реализован в формате Jupyter Notebook и предназначен для демонстрации процессов очистки и анализа данных.

## Основные возможности

### 1. Генерация тестовых данных
Скрипт генерирует CSV-файл `clients_test_data_100.csv`, содержащий 100 записей. Поля включают:
* `client_id`: Идентификатор клиента.
* `full_name`: Случайно сгенерированное ФИО.
* `inn`: ИНН (валидные и намеренно ошибочные для тестов).
* `passport`: Паспортные данные.
* `phone`: Номер телефона в различных форматах.
* `registration_date`: Дата регистрации.

### 2. Инструменты проверки качества (DQ)
В коде реализованы функции для автоматизированного поиска ошибок:
* **Поиск дубликатов**: идентификация повторов по связке ФИО+Паспорт или по ИНН.
* **Валидация ИНН**: проверка длины (10/12 знаков) и расчет контрольного числа.
* **Валидация телефона**: проверка соответствия строгому формату `+7XXXXXXXXXX`.
* **Проверка целостности**: поиск клиентов без привязанных счетов (Referential Integrity).
* **Классификация ошибок**: формирование итогового отчета с перечнем всех найденных нарушений для каждой записи.

## Использование

1.  **Установка зависимостей**:
    Убедитесь, что у вас установлены библиотеки `pandas`, `numpy` и `re`.
2.  **Запуск**:
    Откройте файл `DZ_5.ipynb` в Jupyter Lab, Google Colab или десктопном IDE и выполните ячейки по порядку.
3.  **Результаты**:
    * Сгенерированный файл сохранится по пути `output/clients_test_data_100.csv`.
    * В консоль будут выведены таблицы с найденными нарушениями и дубликатами.
